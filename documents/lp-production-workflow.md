# LP制作ワークフロー

## 概要
このドキュメントは、お客様からLP制作の依頼を受けてから納品までの標準的なワークフローを定義します。

## プラン別対応
- **スタンダードプラン**: 基本的な機能とデザインのLP
- **リッチプラン**: 高度なアニメーションとインタラクティブ要素を含むLP

## 制作フロー

### 1. 初回問い合わせ対応（Day 1）
- [ ] お客様からの問い合わせ受信
- [ ] 24時間以内に初回返信
- [ ] プラン（スタンダード/リッチ）の確認
- [ ] アンケートフォームの送付
- [ ] キャンセルポリシー（テストサイト確認後のキャンセルは1万円）の説明

### 2. 要件収集（Day 1-3）
- [ ] アンケート回答の受領
- [ ] クライアントフォルダの作成 `clients/[client-name]/`
- [ ] アンケート原文を `clients/[client-name]/questionnaire/original.txt` に保存
- [ ] アンケート内容を構造化して `clients/[client-name]/questionnaire/processed.json` に保存
- [ ] `data/clients.json` にクライアント情報を登録（ダッシュボードが自動更新される）
- [ ] ダッシュボード（dashboard.html）で登録確認
- [ ] アンケート内容の整理・分析
- [ ] 不明点の追加ヒアリング

### 3. 仕様書作成（Day 3-5）
- [ ] アンケート内容を基に仕様書作成
- [ ] `clients/[client-name]/specification/` に保存
- [ ] 以下の項目を含む：
  - サイトの目的・ゴール
  - ターゲットユーザー
  - 必要なセクション構成
  - デザインの方向性
  - 必要な機能要件
  - 納期・マイルストーン

### 4. お客様確認（Day 5-7）
- [ ] 仕様書をお客様に提出
- [ ] フィードバック収集
- [ ] 必要に応じて仕様書を修正
- [ ] 最終承認を取得

### 5. デザイン制作（Day 7-14）

#### スタンダードプラン
- [ ] ワイヤーフレーム作成
- [ ] デザインカンプ作成（1案）
- [ ] お客様レビュー・修正（1回）

#### リッチプラン
- [ ] ワイヤーフレーム作成
- [ ] デザインカンプ作成（2案）
- [ ] アニメーション仕様の策定
- [ ] お客様レビュー・修正（2回）

### 6. 開発実装（Day 7-14 ※最速即日可能）

#### スタンダードプラン
- [ ] `clients/[client-name]/development/src/` に開発ファイルを作成
- [ ] HTML/CSS/JavaScript実装
- [ ] レスポンシブ対応
- [ ] 基本的なアニメーション実装
- [ ] フォーム機能実装
- [ ] 開発中のファイルは必ずクライアントフォルダ内で管理

#### リッチプラン
- [ ] `clients/[client-name]/development/src/` に開発ファイルを作成
- [ ] HTML/CSS/JavaScript実装
- [ ] レスポンシブ対応
- [ ] 高度なアニメーション実装（GSAP等）
- [ ] インタラクティブ要素の実装
- [ ] パフォーマンス最適化
- [ ] 開発中のファイルは必ずクライアントフォルダ内で管理

### 7. テスト・品質保証（同日〜Day 15）
- [ ] クロスブラウザテスト
- [ ] モバイルデバイステスト
- [ ] 表示速度測定・最適化
- [ ] SEO基本設定
- [ ] アクセシビリティチェック

### 8. テストサイトデプロイ（Day 15-16）
- [ ] テストサイトへのデプロイ（Vercel/Netlify等）
- [ ] テストURLの発行
- [ ] `clients/[client-name]/test-deployment/` に記録
- [ ] テストサイトURLをお客様に送付
- [ ] 確認期限の設定（通常3営業日）

### 9. お客様確認・承認（Day 16-19）
- [ ] お客様によるテストサイト確認
- [ ] フィードバック受付
- [ ] 承認またはキャンセルの意思確認
- [ ] 承認の場合：承認メールの受領・保存
- [ ] キャンセルの場合：キャンセル料（1万円）の請求

### 10. 納品実施（Day 19-20）
- [ ] 承認メール受領確認
- [ ] 最終ファイルの準備
- [ ] `clients/[client-name]/deliverables/` に格納
- [ ] 納品ファイル一式の送付
- [ ] デプロイメント手順書の提供
- [ ] 運用マニュアルの提供

### 11. 請求・支払い管理（Day 20-27）
- [ ] 請求書の発行
- [ ] 納品後1週間の支払い期限設定
- [ ] 支払い確認
- [ ] 領収書の発行
- [ ] `data/clients.json` のステータス更新

## ファイル管理構造

```
clients/
├── [client-name]/
│   ├── questionnaire/      # アンケート回答
│   │   ├── original.txt    # 原文
│   │   └── processed.json  # 整理済みデータ
│   ├── specification/      # 仕様書
│   │   ├── spec_v1.md
│   │   └── spec_final.md
│   ├── design/            # デザインファイル
│   │   ├── wireframe/
│   │   ├── mockup/
│   │   └── assets/
│   ├── development/       # 開発ファイル
│   │   ├── src/
│   │   └── dist/
│   ├── test-deployment/    # テストサイト情報
│   │   ├── url.txt        # テストURL
│   │   ├── deployed_at.txt # デプロイ日時
│   │   └── approval_status.txt # 承認状態
│   ├── deliverables/      # 納品物
│   │   ├── final_files/
│   │   └── documentation/
│   ├── communication/     # やり取り記録
│   │   ├── emails/
│   │   ├── approval/      # 承認メール
│   │   └── meeting_notes/
│   ├── invoices/          # 請求書
│   └── receipts/          # 領収書
```

## 注意事項

### スタンダードプランの特徴
- 制作期間：最速即日〜約2週間
- デザイン案：1案
- 修正回数：各フェーズ1回まで
- 基本的なアニメーション

### リッチプランの特徴
- 制作期間：約2-3週間
- デザイン案：2案
- 修正回数：各フェーズ2回まで
- 高度なアニメーション・インタラクション
- パフォーマンス最適化

### キャンセルポリシー
- テストサイト確認前：キャンセル料なし
- テストサイト確認後：キャンセル料1万円
- 承認後の納品キャンセル：全額請求

### 支払い条件
- 納品完了後、1週間以内にお支払い
- 銀行振込またはPayPay送金

## コミュニケーション方針
1. 進捗は最低週2回報告
2. 重要な決定事項は必ず文書化
3. お客様からの質問は24時間以内に回答
4. 技術的な説明は分かりやすく簡潔に

## 品質基準
- Lighthouse スコア：全項目80点以上
- 表示速度：3秒以内
- モバイルファースト設計
- アクセシビリティ準拠（WCAG 2.1 Level AA）
